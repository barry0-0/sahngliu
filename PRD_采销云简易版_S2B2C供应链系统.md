## 二、 项目概述与系统定位

### 2.1 项目背景
本系统是基于千匠采销云 S2B2C 大宗商品供应链系统的深度调研，针对我司业务需求进行精简优化后的简易版本。系统保留了核心的交易流程，去除了供应商端、保证金、营销中心、预售集采、授信预存款等复杂模块，形成一个轻量化、易操作的供应链交易平台。

### 2.2 系统定位
**平台定位**：S2B2C 大宗商品供应链交易平台（轻量版）

**核心理念**：
- **去中间件化**：不涉及平台支付功能，只提供结算对账信息
- **统一用户管理**：同一套用户体系，通过用户类型区分商家/买家
- **简化交易流程**：现货合同+竞价合同，保留核心签约流程
- **极简物流**：仅需录入三方物流快递号，无需完整发货流程
- **无退货流程**：交易完成后直接结案

### 2.3 设计原则
1. **轻量化**：移除不必要的模块，保持核心功能完整
2. **易用性**：简洁的交互流程，降低用户学习成本
3. **可扩展性**：预留接口，便于后续功能扩展
4. **一致性**：四端设计风格统一，操作体验一致

## 三、 用户角色与权限体系

### 3.1 用户角色定义

| 角色 | 描述 | 入口 | 权限范围 |
| :--- | :--- | :--- | :--- |
| **平台运营方** | 平台管理员，负责全局监管 | 运营端PC | 数据中心、用户管理、商品审核、财务结算、竞价管理、配置中心 |
| **商家** | 通过审核的卖家企业 | 商家端PC | 店铺管理、商品发布、订单管理、竞价发布、结算对账 |
| **买家/会员** | 平台注册用户，在线采购 | 商城PC/H5 | 商品浏览、下单购买、竞价参与、订单管理、发票申请 |

### 3.2 统一用户模型
```
用户表 (sys_user)
├── user_id          # 用户ID
├── username         # 用户名
├── password         # 密码（加密存储）
├── phone            # 手机号
├── email            # 邮箱
├── user_type        # 用户类型：1-买家/会员  2-商家
├── company_name     # 企业名称
├── business_license # 营业执照
├── credit_code      # 统一社会信用代码
├── contact_person   # 联系人
├── contact_phone    # 联系电话
├── address          # 详细地址
├── status           # 账号状态：0-禁用  1-启用
├── audit_status     # 审核状态：0-待审核  1-审核通过  2-审核拒绝
├── audit_time       # 审核时间
├── create_time      # 注册时间
└── update_time      # 更新时间
```

### 3.3 商家入驻流程
```
1. 用户在商城注册账号（选择用户类型：买家/商家）
2. 买家：注册后直接成为平台会员，可浏览下单
3. 商家：注册后提交资质证明 → 运营端审核 → 审核通过后成为正式商家
4. 商家可登录商家端PC后台进行商品发布、订单管理等操作
```

## 五、 核心业务流程

### 5.1 现货交易流程（简化版）

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│     买家下单      │───▶│   生成订单      │───▶│   合同签署      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                                      │
                                                      ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   买家签收      │◀───│   录入快递号     │◀───│   商家发货      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │
         ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   结算对账      │───▶│   平台返佣      │───▶│   发票开立      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

**详细步骤**：
1. **买家下单**：选择商品，提交订单，确认采购信息
2. **合同签署**：系统生成电子合同（现货合同），买家在线签署
3. **商家确认**：商家确认订单
4. **录入快递号**：商家录入三方物流快递号
5. **买家签收**：买家收到货物，确认签收
6. **结算对账**：平台生成结算对账单，包含返佣信息
7. **平台返佣**：平台按约定比例收取佣金
8. **发票开立**：买家申请开票，平台审核后开立发票

### 5.2 竞价交易流程

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   商家发布竞价   │───▶│   运营审核      │───▶│   竞价公告发布  │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                                      │
                                                      ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   买家报名      │───▶│   竞价出价      │───▶│   竞价结束      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                                             │
         │                                             ▼
         │              ┌─────────────────┐    ┌─────────────────┐
         └─────────────▶│   中标/未中标   │◀───│   出价结果      │
                        └─────────────────┘    └─────────────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │   合同签署      │
                        └─────────────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │   后续流程      │
                        └─────────────────┘
```

**详细步骤**：
1. **商家发布竞价**：商家提交竞价商品信息、底价、竞价规则
2. **运营审核**：运营端审核竞价公告
3. **发布公告**：审核通过后，竞价公告公开发布
4. **买家报名**：有意向的买家提交报名申请
5. **运营审核报名**：运营审核买家报名资格
6. **竞价出价**：买家在竞价时段内出价（支持延时机制：截止前2分钟出价自动延长2分钟）
7. **竞价结束**：最高价中标，生成中标通知
8. **合同签署**：中标后生成竞价合同，买卖双方签署
9. **后续流程**：同现货交易流程（发货→签收→结算→返佣→开票）

### 5.3 询报价流程（沟通版）

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   买家发起询价   │───▶│   商家收到询价   │───▶│   商家报价      │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                     │
         │                     ▼
         │              ┌─────────────────┐
         └─────────────▶│   买家确认      │
                        └─────────────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │   转为订单      │
                        └─────────────────┘
```

**详细步骤**：
1. **买家发起询价**：买家对商品发起询价请求，可上传规格参数
2. **商家收到询价**：商家在询价单列表中看到询价请求
3. **商家报价**：商家根据买家需求提供报价方案
4. **沟通确认**：双方可在线沟通确认细节
5. **买家确认**：买家确认报价，可转为正式订单

┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐

## 六、 合同与返佣体系

### 6.1 合同类型

| 合同类型 | 适用场景 | 签署方 | 主要内容 |
| :--- | :--- | :--- | :--- |
| **现货合同** | 挂牌直销交易 | 买家+商家 | 商品信息、价格、数量、交货期限、交货地点 |
| **竞价合同** | 竞价中标交易 | 买家+商家 | 中标商品、中标价格、交货期限、交货地点 |

### 6.2 合同签署流程
```
1. 订单生成 → 系统生成电子合同 PDF
2. 双方登录各自后台查看合同内容
3. 在线电子签章（CA认证）
4. 合同签署完成，可下载留存
```

### 6.3 返佣机制

| 项目 | 说明 |
| :--- | :--- |
| **返佣主体** | 平台向商家收取交易佣金 |
| **返佣计算** | 返佣金额 = 订单金额 × 返佣比例 |
| **返佣比例** | 可按商品类目、商家等级设置不同比例 |
| **返佣展示** | 返佣信息在订单详情中展示 |
| **返佣结算** | 在结算对账时统一结算 |

## 七、 结算与发票体系

### 7.1 结算中心

| 功能 | 说明 |
| :--- | :--- |
| **结算对账** | 商家查看每笔订单的结算明细，包含订单金额、返佣金额、实收金额 |
| **结算账单** | 按结算周期生成账单，支持月度/季度结算 |
| **结算记录** | 记录每笔结算的状态和历史 |

### 7.2 发票管理

| 功能 | 说明 |
| :--- | :--- |
| **发票信息维护** | 买家维护开票资质（名称、税号、开户行、账号） |
| **申请开票** | 买家对已完成订单申请开具发票 |
| **发票审核** | 运营端审核买家开票申请 |
| **发票开立** | 审核通过后，平台开立发票 |
| **发票查询** | 买家查看已开具的发票 |

### 7.3 票据结算流程
```
1. 买家申请开票（选择订单、发票类型）
2. 运营端审核开票申请
3. 审核通过后开票
4. 发票邮寄/电子发票发送
5. 买家确认收货
```

| **代客下单** | 非核心功能 | 运营代客下单 |

| **预存款** | 不涉及平台支付 | 充值、提现、福点 |
