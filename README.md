# 产业园区商流供应链系统 (S2B2C 轻量版) 产品需求文档 (PRD) V5.0
一、 系统架构与核心交易模型定位
本系统是一个轻量化、去资金池、去复杂物流的产业园区 S2B2C 商流撮合平台。平台不触碰在线支付网关，以“法务电子合同”作为约束纽带，以“对账单开票与抽取佣金”作为盈利核心。
1.1 三大核心交易模型 (Trading Models)
系统支持三种场景的交易流转，最终全部汇聚于标准化的履约与结算体系：
1. 【主流干道】商城现货标准采购：类似传统电商，买家在商城浏览现货 SKU，直接加购下单。
2. 【定制寻源】供需大厅求购匹配：买家发布定制化的采购需求，经平台审核合规后展示在供需大厅。卖家看到后通过系统内置 IM 工具发起沟通、推送报价单，促成转化。
3. 【大宗博弈】竞价拍卖：商家发布拍卖公告，买家在规定时间内集中举牌出价，价高者得（无保证金模式）。
二、 用户角色与权限升维架构 (RBAC)
系统在底层仅有一套用户表（User DB），通过状态标识（User_Type）的切换，实现权限的动态升维。

五、 系统状态机与核心防线约束 (State Machines & Rules)
为确保系统“轻而严谨”，必须在底层流转中设立以下三道红线与一个标准状态机。
5.1 统一交易状态流转机
无论订单来自“商城购买”、“求购议价转化”还是“竞标中标转化”，一旦生成，均必须进入此统一履约管道：

    结算核销中 --> [*]: 运营端查验开票与佣金流水，宣告账单结案

5.2 研发核心防线约束规范
1. 求购信息拦截防线：买家在前端提交的任何求购需求，初始数据库状态必须为 Status = 0 (待审核)。前端【供需大厅】的 API 接口严禁返回及渲染未经过运营端审批的数据，彻底阻断广告与恶意引流。
2. 法务签约物理阻塞防线：在订单流转过程中，若底层数据字段 Contract_Signed = False（即双方未完成电子签约），必须强制置灰（Disable）商家端后台对应的 [发货] 按钮。商流的法务约束优先级必须高于物流动作。
3. 财务解耦计算规范：平台的返佣计算独立于线下真实的银行资金划拨。佣金核算仅在订单跃迁为“已完成”状态时，取当前订单快照的成交总额，乘以系统配置表中读取的当前生效费率（Order\_Amount \times Rate）。结果写入独立结算流水表，确保后期运营修改费率比例时，不影响历史已结账单数据。

附加备注：电子合同与发票税务合规化落地指南
一、 电子合同签署与盖章合规方案
平台自己用代码生成的“电子印章图片”在法律上是无效的。根据《中华人民共和国电子签名法》，合规的电子合同必须通过具备国家国家密码管理局颁发资质的**第三方 CA（数字证书认证机构）**来完成。
● 合规业务流程：
    a. 企业实名认证 (KYC)：商家和买家在入驻平台时，必须调用接口核验企业营业执照、对公账户，以及法定代表人的身份证与人脸识别。
    b. 静默签与意愿签：
        ■ 平台方印章：可以通过后台配置“静默签”，只要订单生成，平台服务协议自动盖上园区的公章。
        ■ 买卖双方印章：订单状态变为“待签合同”时，买卖双方点击签署，系统调用第三方接口，向法定代表人或授权经办人的手机发送短信验证码或拉起刷脸认证（意愿签），验证通过后，CA 机构将数字证书写入 PDF，合同正式生效且不可篡改。
● 建议对接的第三方平台：
    ○ e签宝 或 法大大。这两家是目前国内电子签章的市场份额前两名。
    ○ 对接优势：它们提供非常标准且完善的 Java SDK。在系统架构层面，可以非常平滑、低阻力地嵌入到底层框架中，直接通过微服务调用其 API 完成 PDF 的盖章与存证。
二、 平台抽佣与发票开立税务合规方案
由于平台不触碰在线支付资金（买家直接将货款打给卖家），在税务逻辑上，平台绝对不能就订单的全额货款向买家开具发票，否则会构成“虚开增值税发票”。合规的票务流转必须遵循以下双轨制：
● 合规票税逻辑（双发票流）：
    a. 货款发票（卖家 \ 买家）：买家线下支付了 10 万元货款给卖家，卖家有义务向买家开具 10 万元的货物销售增值税发票。
    b. 佣金发票（平台 \ 卖家）：平台按 2% 收取了 2000 元的抽点。这 2000 元是平台实际的营业收入，**平台（园区）**需要向卖家开具 2000 元的“信息技术服务费”或“平台服务费”增值税发票。
● 平台系统实现方式：
    ○ 发票中心改造：买家在前端“申请开票”时，系统应提示该货款发票由对应商家开具。系统作为信息桥梁，将开票请求推送给商家。
    ○ 平台佣金结算：每月生成的《结算对账单》就是平台向商家收款和开具服务费发票的账单依据。
● 建议对接的第三方平台：
    ○ 百旺金赋 或 诺诺发票。它们全面支持国家税务总局最新的“全面数字化的电子发票（数电票）”。
    ○ 对接优势：通过其开放 API，当月底《结算对账单》核销确认后，系统可以自动一键请求诺诺/百旺的接口，秒级开出数电票并把 PDF/OFD 凭证回传到商家后台。
三、 业务升维价值总结
在系统初期就将合规的 CA 确权和真实的数电票体系打通，意义不仅在于防范税务风险。更重要的是，一旦实现了业务流、合同流、发票流的三流验证，这套平台沉淀的数据就具备了极高的金融信用价值。未来如果产业园区需要基于这些真实交易数据去延展供应链金融业务（例如针对非标大宗物资的仓单质押、或者是订单融资），这套系统将直接成为最坚实的底层信任基石。
