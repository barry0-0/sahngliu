# 《User Story Mapping》提炼存档

原著：Jeff Patton,《User Story Mapping: Discover the Whole Story, Build the Right Product》(O'Reilly, 2014)，序言 by Marty Cagan & Alan Cooper。

提炼四原则：忠于原著 / 可执行 / 可追溯 / 有边界。

## 章节 → 产出映射
- 第1章 &「The Word Is Not the Thing」→ 共享理解优先于文档；故事=促成对话。
- 第2章 建图基础 → backbone/骨架概念。
- 第5章 the map → 二维地图（横轴叙事流程/纵轴优先级）；扁平清单反模式。
- 第6章 slicing → walking skeleton；横切发布 vs 纵切模块。
- 第13章 → epic 拆分为可交付切片。
- 第14章 → 细节随临近开发再展开（不追求写全）。
- 第15章 → 用户故事 Connextra 模板（作为<谁>/想<做什么>/以便<价值>）。
- 第3章 outcome → 关注用户行为/业务结果变化而非仅输出。

## 收录判断
- 可执行 → 收录：建骨架步骤、横切切片、共享理解协作、拆 epic、故事模板、健康地图清单。
- 背景注释（不进规则）：作者项目轶事、卡片墙工具细节。

## 边界
- 2014 出版，方法稳定；仅解决"已决定要做→如何结构化表达与切分"。
- 无清晰用户旅程（算法/基础设施）时地图形态需变通。
- 假设团队能协作建图；远程/话语权不对等需额外机制。
- 与套件分工：Story Mapping = 结构化/切分；Mom Test = 需求真假；Cagan/build-trap = 该不该做。

（本存档基于原著框架与公认章节结构提炼；个别规则章节号若有出入以原书为准。）
