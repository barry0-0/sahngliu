# 产品经理顾问团 · PM Advisory Suite

> 由 [career-skill-factory](https://github.com/SpaceZephyr/career.skill) 蒸馏生成。
> 方向：**通用 / 综合产品经理**　侧重：**提升产出质量**　调研截止：**2026-07-09**

一支给产品经理用的私人顾问团：3 位专家负责心智模型、3 本经典负责可执行方法论、1 个总控负责路由与评审会。它擅长把一个需求/方案从"感觉不错"打磨到"证据扎实、结构清晰、连回成效"。

## 包含什么（7 个 Skill）

| Skill | 类型 | 核心镜片 | 什么时候用 |
|------|------|---------|-----------|
| `pm-advisor-cagan` | 专家视角 | 四大风险 + 赋能团队 + discovery 先于 delivery | 这需求该不该做、团队该怎么组、风险在哪 |
| `pm-advisor-torres` | 专家视角 | 持续发现 + 机会解决方案树 + 假设验证 | 怎么做用户发现、需求连回 outcome、先验哪个假设 |
| `pm-advisor-yujun` | 专家视角 | 用户价值公式 + 交易模型 + 决策效用 | 这功能价值多大、用户为何不迁移、取舍怎么算账 |
| `pm-method-mom-test` | 方法论 | 三条铁律 + 谈过去不谈未来 | 访谈设计、提真信号、判断需求真假 |
| `pm-method-story-mapping` | 方法论 | 故事地图骨架 + 横切发布切片 | 需求结构化、切 MVP、跨团队对齐 |
| `pm-method-build-trap` | 方法论 | 功能陷阱自检 + 战略部署 + 产品 kata | 判断功能工厂、战略落成效、系统性验证 |
| `pm-advisory-board` | 顾问团总控 | 自动路由 + 分歧地图 + 评审会纪要 | 多视角评审、不知道该找谁时的入口 |

## 覆盖的高频决策/产出场景

需求真伪判断 · 立项该不该做 · 用户为何不迁移 · 用户发现与访谈 · 需求结构化与 PRD 全貌 · MVP 切片 · 功能工厂自检 · 战略落到成效 · 方案假设验证 · 优先级排序。

## 怎么用

1. **不知道找谁** → 先装并触发 `pm-advisory-board`，描述你的问题，它会推荐 1-2 个成员；说"都上"可开评审会。
2. **单专家提问** → `用 Cagan 的视角看这个需求` / `Torres 会怎么做这个发现` / `俞军怎么算这个功能的用户价值`。
3. **方法论过一遍** → `用 Mom Test 帮我改访谈提纲` / `用故事地图拆这个需求` / `用功能陷阱自检看看我们团队`。
4. **开评审会** → `让专家们评审这个方案` / `都上` → 并行输出共识 / 分歧 / 综合结论。

一条典型链路（提升产出质量）：Mom Test 取真信号 → Torres 挂进机会树、验最风险假设 → 俞军算用户价值与替换成本 → Story Mapping 排骨架切 MVP → Build Trap 确认连回 outcome → Cagan 复核四大风险与团队赋能。

## 安装

每个子目录都是一个独立 Skill（含 `SKILL.md`，专家/方法论另含 `references/`）。基于开放的 Agent Skills 协议，可在 Claude Code、Cursor、Codex 等兼容 runtime 使用：

- **整套**：把 `pm-advisory-suite/` 下 7 个子目录复制到你的 skills 目录。
- **按需**：先装 `pm-advisory-board` 体验路由，再按需装单个专家/方法论。
- 若发行方支持 `.skill` 安装包，可将任一子目录打成 zip 后改名 `.skill` 安装。

## 阵容说明（为什么是这三位 + 三本）

- **互补而非堆名气**：Cagan 管"组织怎么赋能团队交付正确的东西"，Torres 管"怎么持续发现、用证据决策"，俞军管"用户价值怎么量化、迁移成本怎么算"——一个组织视角、一个方法视角、一个经济视角。
- **中西并重**：两位西方现代产品学派代表 + 一位中文大厂语境的产品理论家。
- **书补专家没讲透的操作层**：Mom Test 给逐句访谈话术，Story Mapping 给结构化与切片，Build Trap 给战略—成效对齐——都是"专家谈理念、书给你能直接照做的步骤"。

## 诚实边界

- 所有专家视角基于公开信息蒸馏，**≠ 本人真实判断**；三位专家均在世，观点会更新，本套件是 2026-07 的历史快照。
- 方法论 Skill 只做"框架 + 规则 + 章节引用"，不复制原文大段内容，版权归原作者，仅供个人学习。
- 本团同属"以用户价值为中心的现代产品学派"，视角互补但同源——**缺一个纯增长/纯商业化/纯数据分析视角**，这是刻意聚焦"产出质量"的结果，也是局限。
- 不覆盖：增长投放、商业化定价、实验统计、AI/大模型产品的技术选型（见仓库自带「AI 产品经理」套件）、中国特定合规、B 端复杂销售链路落地。
- 顾问团输出是"三位聪明朋友给你一下午的建议"，不是最终答案。重大决策请结合一手信息自行判断。

## 更新建议

行业方法论相对稳定，但专家近 12 个月的新表达（尤其 Cagan《Transformed》后的产品运营模型、Torres 的发现实践演进）值得定期补研。想更新某一位：对该成员重跑调研、增量更新其 Skill 与总控分歧地图即可，无需重做全套。
