# 《Escaping the Build Trap》提炼存档

原著：Melissa Perri,《Escaping the Build Trap: How Effective Product Management Creates Real Value》(O'Reilly, 2018)。全书结构分四部分。

提炼四原则：忠于原著 / 可执行 / 可追溯 / 有边界。

## 部分 → 产出映射
- 第一部分 The Build Trap → 功能陷阱定义；output vs outcome；功能工厂反模式；功能陷阱自检框架。
- 第二部分 The Role of the Product Manager / Product Kata → PM=价值交换管理者（非项目经理）；产品 kata（方向/现状/目标/障碍/实验循环）。
- 第三部分 Strategy → 战略部署四层（愿景/战略意图/产品行动/选项）；路线图≠承诺清单；好战略=决定不做什么。
- 第四部分 Product Process / Organization → 可复制的产品流程与组织能力，反"英雄式 PM"。

## 收录判断
- 可执行 → 收录：功能陷阱自检、战略部署四层、产品 kata、检查清单、七条规则。
- 背景注释：书中 Marquetly 虚构公司案例仅作说明，不进规则。

## 与套件分工（防重叠）
- 与 Cagan「产品运营模型」高度共鸣 → 本书出操作层（战略部署四层 + product kata），Cagan 出理念与团队赋能。评审时二者互为印证，不重复堆叠。
- 与 Torres：Torres 的 assumption test 是 product kata"实验"步的具体手法 → 可衔接。

## 边界
- 2018 出版，方法稳定；诊断组织/战略层做无用功，不含访谈/交付细节。
- 依赖可定义度量的成效；成效难量化场景需谨慎。
- 早期小团队四层战略可精简。

（本存档基于原著四部分结构与核心框架提炼；若个别归属按二手细节有出入，以原书为准。）
