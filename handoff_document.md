# 享宇森云 - 产业园区商流供应链系统 (S2B2C 轻量版) 项目交接文档 (Handoff Document)

本项目是一个轻量化、去资金网关、以“法务电子合同”为约束纽带、以“线下履约对账及抽佣”为核心的**大宗商品供应链商流平台高保真原型系统**。

## 🔄 2. 系统核心数据流动与联动机理

项目采用 **纯前端无状态高保真架构**，所有数据的存储与变更全部流经 `platform/assets/js/mock-data.js` 的 `MockData` 总线：
1. **多端数据互通**：如运营端关停店铺后，买家端商城与商家端后台状态实时失效；商家端修改并提交申诉后，商家状态恢复；竞价大厅出价与定标后，自动在订单中心生成“待签约”大宗合同订单。
2. **三方解耦设计**：无在线支付接口，订单履约链条为：`创建 -> 双边电子签约盖章 -> 线下公对公汇款确认 -> 卖家录入物流发货 -> 买家确认收货结算`。

## ✨ 3. 已完成的关键技术改造与亮点

### 3.1 订单履约详情与履约轨迹（五端同步）
* **动态响应式设计**：在 `components.js` 中集成了 `UI.showOrderDetail(orderId)` 通用方法。
  * **PC 端三端**：点击列表中的订单 ID 或最右侧新增的“详情”按钮，将弹起精美的弹窗，图示化展示包含“各方签约时间、付款时间、发货时间、结清完结时间”的纵向履约跟踪轨迹。
  * **H5 端双端**：点击移动订单卡片自动拉起响应式的下半屏 Bottom Sheet 详情页。
* **卡片细节微调**：移除了“单号：”前缀，强化了货品字号（`15px`），精简了店铺字号（`11px`），去除了重复的状态标签。

### 3.2 商家管理端（PC & H5）
* **数据中心升级**：PC 商家后台首个默认页面升级为**数据中心**，以条柱图表直观展示“品类销售占比”及“采购商交易频次与总额排行”，搭配 GMV、待发货单数等 KPI 磁贴。
* **满屏极简风格**：移除了店铺装修卡片的 `800px` 宽度瓶颈，改为全屏 Grid 双列响应式布局。
* **业务状态对齐**：当店铺被平台处罚下架时，PC/H5 同步显示醒目的红字“违规下架公示”与原因，允许重新编辑商户名一键申请复核上架。
* **销售模式选择**：商家发布/查看商品时，表格与属性中完美增加并透出 **“现货”** 与 **“预售”** 模式的选择。

### 3.3 运营端后台
* **上架商品管理（原商品审核）**：
  * 重命名了菜单与视图，在货品列表中引入 **“主图”** 与 **“类型标签（现货/预售）”** 的展示。
  * 引入了 **“编辑商品信息”弹窗**，运营人员可以直接编辑商品的名称、单价、品类分类、库存、销售模式以及商品图片 URL/本地模拟上传，极大地方便了平台监督纠偏。

### 3.4 买家商城端（PC & H5）
* **个人中心侧边栏美化**：优化了 [mall.html](file:///Users/barry/Desktop/工作/享宇森云/商流/platform/mall.html) 个人中心左侧菜单，头像区域改为蓝紫向炫酷渐变搭配发光阴影，菜单条目采用全新圆角气泡选项式高亮及 Hover 动效。
* **微信小程序式 Header (H5)**：买家端与卖家端 H5 顶部均加设了高仿真的微信小程序**胶囊按钮 (`...` 与 `o`)**，且商城首页首次加载时自动隐藏返回箭头，跳转后智能呈现。

## 🚀 4. 交接运行与二次开发指南

1. **零依赖运行**：项目纯原生编写，无需配置 `npm install`、`vite` 等环境。直接双击 `platform/` 目录下的任何 `.html` 即可在主流浏览器中体验全功能模拟。
2. **二次开发**：
   * **新增状态节点**：可直接在 `mock-data.js` 的 `MockData.orders` 中修改 status 状态定义。
   * **样式修改**：大宗商品的通用元素配置在 `platform/assets/css/global.css` 中，个人中心及移动端的独立样式分别放置在 `mall.css` 与 `h5.css`。
