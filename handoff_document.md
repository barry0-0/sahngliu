# 咖喱粑粑 - 产业园区商流供应链系统 (S2B2C 轻量版) 项目交接文档 (Handoff Document)

本项目是一个轻量化、去资金网关、以“法务电子合同”为约束纽带、以“线下履约对账及抽佣”为核心的**大宗商品供应链商流平台高保真原型系统**。目前整体命名品牌为 **“咖喱粑粑”**。

---

## 📂 1. 项目目录结构与技术栈

项目采用 **纯前端原生架构 (HTML5 + CSS3 + Vanilla JS)**，无需任何构建工具或依赖安装（零依赖），直接在浏览器中打开即可运行。

```text
├── platform/                          # 系统高保真原型核心目录
│   ├── mall.html                      # 【买家端】PC 商城（大宗交易、求购大厅、竞价大厅、个人中心）
│   ├── h5.html                        # 【买家端】H5 移动端（小程序化首页、购物车结算、电子合同签约、个人中心）
│   ├── merchant.html                  # 【商家端】PC 管理后台（数据中心、店铺装潢、商品/上架管理、订单/竞价管理）
│   ├── merchant-h5.html               # 【商家端】H5 移动后台（店铺指征、商品管理、双通道订单中心、移动签约）
│   ├── admin.html                     # 【平台运营端】园区总控后台（BI看板、店铺关停监管、上架商品管理、求购/合同审核）
│   └── assets/                        # 静态资源与前端核心代码
│       ├── css/                       # 样式表（global.css, admin.css, mall.css, h5.css）
│       └── js/                        # 业务逻辑
│           ├── components.js          # 全局 UI 核心组件（Toast 提示、Modal 弹窗、Tab 切换、通用订单履约轨迹详情弹窗）
│           ├── mock-data.js           # 统一数据总线（核心 Mock 数据库，实现多端数据同步与真实业务联动）
│           ├── mall.js                # 买家端 PC 业务逻辑
│           ├── h5.js                  # 买家端 H5 业务逻辑
│           ├── merchant.js            # 商家端 PC 业务逻辑
│           ├── merchant-h5.js         # 商家端 H5 业务逻辑
│           └── admin.js               # 运营端后台业务逻辑
```

## 🔄 2. 系统核心数据流动与联动机理

项目所有数据的存储与变更统一流经 `platform/assets/js/mock-data.js` 中的 `MockData` 对象：
1. **多端状态联动**：由于是单页面共用同一内存数据库（或跨页面通过 localStorage 同步），各角色端状态是实时的。例如：运营端处罚关停某店铺后，买家端商城中该店铺的商品会自动下架失效，商家后台会同步提示处罚并限制操作。
2. **非资金网关履约模式**：系统不接入第三方支付，而是依靠法务合同进行双边强约束。履约链条为：`意向/竞价成交 -> 生成电子合同 -> 双方在线电子盖章 -> 线下公对公汇款对账 -> 卖家发货录入物流 -> 买家确认收货结算`。

## 🛠️ 3. 系统核心功能模块规格说明

### 3.1 订单履约详情与轨迹跟踪 (Order Fulfillment Tracking)
* **业务轨迹可视化**：集成在 `components.js` 的 `UI.showOrderDetail(orderId)` 方法中。
  * **PC 端**：点击订单编号或“详情”按钮，弹起模态框，直观展示纵向订单轨迹（包括合同签署时间、付款完成时间、物流发货时间、结清完结时间）。
  * **H5 端**：采用响应式的底部滑出面板 (Bottom Sheet) 展现完整的履约追踪信息。
* **信息展示优化**：订单卡片排布紧凑，高亮货品字号（`15px`），弱化辅助性店铺信息（`11px`），状态标签展示单一清晰。

### 3.2 商家管理后台 (Merchant Panel)
* **数据中心 (BI 看板)**：提供可视化的图表统计。左侧是“品类销售占比”的环状/柱状比对，右侧是“采购商交易频次与总额排行”，配合 GMV 总额、待处理订单等核心 KPI 指标。
* **大盘看板指标简化**：移除了不必要的环比增幅标识和目标率说明，删除了右上角导出动作，使整体信息结构聚焦于核心销售指标。
* **经营状态与申诉**：支持展示违规处罚状态。当店铺被公示下架时，后台主页展示红字警告及处罚原因，并提供修改商户名称一键申请复核的申诉入口。
* **销售模式发布**：在发布与管理商品时，系统原生支持选择并透出 **“现货”** 或 **“预售”** 两种销售属性。
* **页面布局**：店铺装潢及商家后台主体视图采用响应式 Grid 布局，具备全屏适配的宽屏浏览体验。

### 3.3 平台运营端总控台 (Platform Admin)
* **上架商品管理**：列表显示商品的缩略主图和“现货/预售”类型标签。运营人员可以在此对商品的名称、单价、品类分类、库存、销售模式等基础属性进行直接编辑与核校，提供平台层面的监管纠偏能力。
* **店铺关停与审核**：包含对商家违规处罚的关停机制以及合同、求购单的后台审核链条。

### 3.4 买家商城端 (Buyer Mall)
* **PC 端个人中心**：左侧菜单 `#uc-menu` 结构化分类为“设置”（含密码修改）、“个人记录”（个人/企业认证、协议管理、关注列表）以及“交易市场”（订单管理、发票申请、求购记录），条目带有高亮气泡选项与 Hover 交互特效。
* **H5 端页面架构**：
  * **底部导航栏**：Tab 项目配置为：“首页” -> “竞价” -> “找货源” -> “消息” -> “我的”。
  * **小程序式头部**：所有 H5 页面顶部右侧均集成有高仿真胶囊按钮 (`...` 选项与 `o` 退出)。首页自动隐藏返回键，内层页面智能展示返回箭头。
  * **列表渲染**：个人中心内集成了“我的求购记录”（求购寻源状态、报价单数量统计）和“发票中心”（发票类型、发票金额、关联订单、邮寄状态）的动态模拟渲染与交互操作。

### 3.5 双通道合同签署与货款支付机制 (Contract & Payment Channels)
* **合同签署双通道 (买卖双边支持)**：在 PC/H5 各自的个人订单中心或商家后台订单管理中，点击签署合同时支持以下两种路径：
  - **线上电子签章**：使用系统模拟的 CA 电子证书及印章（采购章/供应章）在线快速完成签名盖章锁定。
  - **纸质合同文件上传**：支持将双方线下签字盖章后的合同照片或扫描件 (PDF/JPG/PNG) 上传导入，自动记录为订单绑定的物理存证文件（如 `order.contractFile`），适应线下签约需求。
* **付款结算双通道 (买家端支持)**：待付款状态的订单支持以下支付模式：
  - **在线快捷支付**：买家通过关联的快捷虚拟资金账户进行一键在线扣款及账务清算。
  - **线下打款凭证上传**：提供平台对公收款账户银行信息（开户行、账号、户名等），支持买家在线下完成银行汇款后上传转账底单/银行回执单文件 (Image/PDF) 存证（如 `order.paymentVoucher`），满足公对公转账结算模式。

## 🚀 4. 运行与二次开发指南

1. **运行要求**：直接双击 `platform/` 目录下的 HTML 文件即可在浏览器中查看高保真原型。
2. **状态定制**：若需要拓展或修改初始数据，可直接修改 `platform/assets/js/mock-data.js` 中 `MockData` 对象的初始化数组（例如 `orders`、`products`、`users` 等）。
