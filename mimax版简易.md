# 商流供应链系统 S2B2C 产品需求文档 (PRD)
---
## 一、项目概述与定位
### 1.1 项目背景
本系统是面向大宗商品供应链交易的 S2B2C 平台，整合运营端、商家端、买家商城端（H5+PC）四端，实现商品展示、在线交易、合同签署、竞价拍卖、结算对账、发票管理等全链路功能。
### 1.2 产品定位
**核心定位**：极简版 S2B2C 大宗商品供应链交易平台
**产品特色**：
- 统一用户体系（会员+商家双角色，审核入驻）
- 现货交易 + 竞价交易双模式
- 合同签署与订单关联
- 平台结算与返佣
- 发票开立与票据结算
- 无供应商端、无保证金、无支付功能

## 三、用户角色与权限体系
### 3.1 用户角色
| 角色 | 说明 | 入口 |
|------|------|------|
| 会员/买家 | 商城注册用户，可在商城下单采购 | 商城PC、H5 |
| 商家 | 会员申请入驻并通过审核，可在商家端发布商品、管理订单 | 商家端PC |
| 运营管理员 | 平台运营方，全局管控 | 运营端PC |
### 3.2 统一用户体系
- **注册**：通过商城进行会员注册
- **入驻**：商家提交入驻申请 → 运营端审核 → 通过后获得商家权限
- **权限控制**：同一用户表，通过 `userType` 和 `businessStatus` 区分普通会员和商家

## 六、合同与结算体系
### 6.1 合同类型
| 合同类型 | 触发场景 | 说明 |
|---------|---------|------|
| 现货合同 | 现货订单确认 | 买家下单后自动生成，含商品明细、价格、交货条件 |
| 竞价合同 | 竞价中标确认 | 竞价成交后生成，含成交价、交割时间等 |
### 6.2 合同内容要素
- 甲方（买家）/ 乙方（商家）信息
- 商品明细（名称、规格、数量、单价、总额）
- 交货地点与方式
- 付款方式（线下自行转账）
- 违约责任
- CA电子签章
### 6.3 结算中心
#### 6.3.1 结算对象
- 商家结算：平台与商家之间的货款结算
- 平台返佣：平台从每笔交易中抽取佣金
#### 6.3.2 结算单据
| 单据类型 | 说明 |
|---------|------|
| 结算单 | 订单完成后生成，含商品明细、货款、佣金、实结金额 |
| 返佣单 | 记录平台返佣金额 |
| 付款单 | 平台向商家付款的记录 |
| 退款单 | 退款申请与处理 |
#### 6.3.3 结算流程
订单完成 → 结算单生成 → 佣金计算 → 应付商家金额 = 订单金额 - 佣金 → 商家提现 → 运营端付款

*.txt
Plaintext
### 6.4 返佣体系
| 字段 | 说明 |
|------|------|
| 订单金额 | 商品总价 |
| 平台佣金比例 | 由运营端配置（按商品类目或商家等级）|
| 佣金金额 | 订单金额 × 佣金比例 |
| 商家实结金额 | 订单金额 - 佣金金额 |

## 八、竞价体系
### 8.1 竞价角色
- **发起方**：商家（发布竞价资源）
- **参与方**：买家（会员）报名参与竞价
- **监管方**：运营端（审核竞价公告、管理场次）
### 8.2 竞价资源
- 商家提交竞价商品资源
- 运营端审核并发布竞价公告
- 含：底价、加价幅度、竞价开始/结束时间、看货时间等
### 8.3 竞价规则
- 竞价时段内买家可多次出价
- 当前最高价实时更新
- 竞价结束前2分钟出价自动延时
- 竞价结束后最高价者中标
### 8.4 竞价场次
- 竞价预告：提前发布竞价信息
- 竞价公告：正式竞价场次
- 竞价资源：可交易的商品资源
